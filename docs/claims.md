# Claims

Claim status is regenerated by `scripts/run_claim_audit.sh` into `results/claims_status.md` and `results/claims_status.json`. This document states the intended claim boundary.

## Theorem Claims

- SUPPORTED: The exact finite tie-aware Best-of-N law characterizes expected selected real utility for a fixed finite candidate pool and scorer.
- SUPPORTED: The same law handles real-valued utility, binary utility, tied scores, constant utility, oracle scores, and anti-aligned scores.

## Controlled VLA-Style Toy Claims

- SUPPORTED if artifacts exist: controlled VLA-style scenes can show semantic affordance over-selection, where selected semantic score rises with `N` while physical utility saturates or drops and physical violations rise.
- PARTIAL by design: these are controlled CPU toy scenes, not full robotics benchmarks.

## Learned VLA-Style Claims

- SUPPORTED if artifacts exist: a learned language-conditioned scorer consumes instruction features, visual/object observation features, and action candidate features, and its semantic score is evaluated separately from physical utility.
- SUPPORTED if artifacts exist: raw learned semantic scoring can show high-N regret, and calibrated or grounded scoring can improve selected-tail real utility.
- SUPPORTED if artifacts exist: a heavier PyTorch semantic scorer reproduces the selected-tail failure and its calibrated variant repairs utility in controlled settings.

## Semantic Distractor/Object-Binding Claims

- SUPPORTED if artifacts exist: distractor scenes measure wrong-object, wrong-target, unreachable, collision, fragile, and stability failures.

## Rendered Visual / Simulator Claims

- SUPPORTED if artifacts exist: rendered visual scenes and simulator-style utility reproduce semantic affordance over-selection.
- PARTIAL by design: rendered scenes and the simulator-style evaluator close part of the symbolic-to-visual gap, but they are still controlled evidence.

## Grounding/Calibration Repair Claims

- SUPPORTED if artifacts exist: physical verification, combined scoring, or pilot-label calibration can improve high-N selected utility in controlled VLA-style settings.
- PARTIAL: this does not imply every physical verifier or calibrator will work.

## Noisy Verifier And Calibration Robustness Claims

- SUPPORTED if artifacts exist: noisy verifiers and calibration budget/noise stress tests show repair can be partial or unsafe under imperfect grounding.
- SUPPORTED if artifacts exist: the deployment gate can continue to block high-N when noisy repair fails to reduce violations enough.

## Optional Benchmark Claims

- UNSUPPORTED: real OpenVLA, RT-style, LIBERO, or robot benchmark validation is not claimed in v1 unless a guarded adapter is implemented and run.
- SUPPORTED: optional SmoLVLA adapter status can be recorded with a precise run-or-skip reason.

## Unsupported Future Robotics Claims

- UNSUPPORTED: real-robot validation.
- UNSUPPORTED: full robot foundation model validation.
- UNSUPPORTED: universal deployment recipe.

## Forbidden Claims

The repository must not assert these as true:

- "We prove VLAs work."
- "We solve robot planning."
- "We validate on real robots."
- "This is a universal VLA inference recipe."
- "Best-of-N always helps."
- "Grounding always fixes VLA errors."
- "This is full robot foundation model validation" unless actual models and benchmarks are used.
- "This is not toy evidence" unless real benchmarks are implemented.
