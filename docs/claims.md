# Claims

Claim status is regenerated by `scripts/run_claim_audit.sh` into `results/claims_status.md` and `results/claims_status.json`. This document states the intended claim boundary.

## Theorem Claims

- SUPPORTED: The exact finite tie-aware score-tail law characterizes expected selected real utility for a fixed finite candidate pool and scorer.
- SUPPORTED: The same law handles real-valued utility, binary utility, tied scores, constant utility, oracle scores, and anti-aligned scores.
- SUPPORTED: semantic scores alone cannot certify high-N physical utility when upper-tail physical utility is unconstrained; this is covered by no-free-lunch tests and theory docs.

## Exact-Law Validation Claims

- SUPPORTED by current artifacts: every main experiment compares exact finite-law selected utility with Monte Carlo selected utility and confidence intervals.
- SUPPORTED by current artifacts: exact-law prediction error remains small across controlled, learned, distractor, repair, rendered, and robustness summaries.

## Controlled VLA-Style Toy Claims

- SUPPORTED by current artifacts: controlled VLA-style scenes can show semantic affordance over-selection, where selected semantic score rises with `N` while physical utility saturates or drops and physical violations rise.
- BOUNDARY: these are controlled CPU toy scenes, not full robotics benchmarks.

## Learned VLA-Style Claims

- SUPPORTED by current artifacts: a learned language-conditioned scorer consumes instruction features, visual/object observation features, and action candidate features, and its semantic score is evaluated separately from physical utility.
- SUPPORTED by current artifacts: raw learned semantic scoring can show high-N regret, and calibrated or grounded scoring can improve selected-tail real utility.
- SUPPORTED by current artifacts: a heavier PyTorch semantic scorer reproduces the selected-tail failure and its calibrated variant repairs utility in controlled settings.

## Semantic Distractor/Object-Binding Claims

- SUPPORTED by current artifacts: distractor scenes measure wrong-object, wrong-target, unreachable, collision, fragile, and stability failures.

## Rendered Visual / Simulator Claims

- SUPPORTED by current artifacts: rendered visual scenes and simulator-style utility reproduce semantic affordance over-selection.
- BOUNDARY: rendered scenes and the simulator-style evaluator close part of the symbolic-to-visual gap, but they are still controlled evidence.

## Grounding/Calibration Repair Claims

- SUPPORTED by current artifacts: physical verification, combined scoring, or pilot-label calibration can improve high-N selected utility in controlled VLA-style settings.
- BOUNDARY: this does not imply every physical verifier or calibrator will work.

## Certified TailGuard Method Claims

- SUPPORTED by current artifacts: Certified TailGuard applies hard modeled-physics certificates before tail calibration and returns exactly one of `allow_high_n`, `stop_early`, `collect_pilot_labels`, or `block_high_n`.
- SUPPORTED by current artifacts: constructed gate examples exercise all four decisions with matching reason codes.
- SUPPORTED by current artifacts: high-N is allowed only when certified selected-tail lower confidence bounds clear both the `N=1` and random-selection baselines by a margin.
- SUPPORTED by current artifacts: controlled Certified TailGuard utility is at least `0.98` and certified violation is at most `0.01` in the main controlled stress setting.
- SUPPORTED by current artifacts: public result fields include `certificate_pass`, `certificate_failure_types`, `certified_candidate_count`, `fallback_used`, `abstention_reason`, `certified_selected_utility`, and `certified_violation_rate`.
- BOUNDARY: this is an inference-time controller for controlled candidate pools, not a universal robot deployment recipe.

## Certified TailGuard Ablation And Failure-Honesty Claims

- SUPPORTED by current artifacts: ablations cover the physical certificate, verifier score, pilot labels, empirical lower bound, adaptive `N`, and abstention/fallback.
- SUPPORTED by current artifacts: each named removal fails the controlled acceptance threshold in at least one component-specific stress regime while the full method passes.
- SUPPORTED by current artifacts: some regimes explicitly fall back or abstain instead of claiming repair.

## Phase Diagram Claims

- SUPPORTED by current artifacts: phase diagrams sweep semantic/physical misalignment, distractor salience, obstacle density, hidden constraints, verifier false positives/negatives, and label noise.
- SUPPORTED by current artifacts: the phase diagrams record utility, violation, confidence, label-budget, and gate outcomes across stress regions.

## Calibration Sample Complexity Claims

- SUPPORTED by current artifacts: pilot-label budget/noise curves show lower-bound tightening and the tradeoff between scarce labels, noisy labels, and gate decisions.
- SUPPORTED by current artifacts: scarce selected-tail evidence triggers `collect_pilot_labels`.

## First-Principles Physics Claims

- SUPPORTED by current artifacts: geometry-based reach envelopes, swept-volume collision, receptacle compatibility, stability margin, fragile/heavy handling, and blocked paths are evaluated.
- SUPPORTED by current artifacts: adversarial stress families include hidden obstacles, semantic-tail verifier false positives, partial observation, object identity spoofing, wrong high-plausibility receptacles, fragile/heavy ambiguity, correlated pilot-label noise, train/pilot/test shift, and low certified-candidate-count scenes.
- SUPPORTED by current artifacts: stress artifacts report raw utility, Certified TailGuard utility, violation, abstention, fallback, gate distribution, and certificate failure decomposition.
- BOUNDARY: these checks are controlled simulator evidence, not real robot dynamics.

## Noisy Verifier And Calibration Robustness Claims

- SUPPORTED by current artifacts: noisy verifiers and calibration budget/noise stress tests show repair can be incomplete or unsafe under imperfect grounding.
- SUPPORTED by current artifacts: the deployment gate can continue to block high-N when noisy repair fails to reduce violations enough.

## Optional Benchmark Boundary Claims

- SUPPORTED boundary claim: real OpenVLA, RT-style, LIBERO, or robot benchmark validation is not implemented and is not claimed unless a guarded adapter is implemented and run.
- SUPPORTED: optional SmoLVLA adapter status can be recorded with a precise run-or-skip reason.
- SUPPORTED by current artifacts: the latest cached SmoLVLA policy probe can load locally on CPU and emit an action chunk for synthetic visual/state/language input.
- SUPPORTED by current artifacts: the optional rendered-input bridge status is recorded with `action_decode_supported: false` and `physical_success_claimed: false` unless an action decoder is implemented.
- SUPPORTED by current artifacts: external integration, benchmark reset/step/reward artifacts, exposed success metrics, physical outcome claims, and TailGuard method outcome claims are separate claim levels.
- SUPPORTED boundary claim: a RoboCasa or LIBERO integration artifact alone does not imply benchmark stepping, success metrics, physical outcome evidence, or TailGuard method evidence.
- BOUNDARY: the SmoLVLA inference probe is real model plumbing, not LIBERO/OpenVLA benchmark validation and not physical success evaluation.

## Future Robotics Boundary Claims

- SUPPORTED boundary claim: real-robot validation is not established and is not claimed.
- SUPPORTED boundary claim: full robot foundation model validation is not established and is not claimed.
- SUPPORTED boundary claim: a universal deployment recipe is not established and is not claimed.

## Forbidden Claims

The repository must not assert these as true:

- "We prove VLAs work."
- "We solve robot planning."
- "We validate on real robots."
- "This is a universal VLA inference recipe."
- "score-tail always helps."
- "Grounding always fixes VLA errors."
- "This is full robot foundation model validation" unless actual models and benchmarks are used.
- "This is not toy evidence" unless real benchmarks are implemented.
